# 기타 (느낀 점, 후기)

## 이은석
- 미리 프로그램은 기획하고 모델링을 확실하게 해두는게 중요하다고 생각
  - 원하는 데이터를 뽑을 때 vue에서 store에서 돌고 돌아서 겨우 뽑는 경우가 발생함
  - 백엔드가 데이터를 미리 정리해서 깔끔하게 주는 것이 프론트에게도 훨씬 좋을 것으로 보였음
  - 다음에는 완전 서로 100% 프론트 백을 담당해서 프로그래밍 해보고 싶어졌음 

- 코드가 길어지니 점점 보기가 어려워지는 상황 발생
  - 모듈로 나눠서 관리하면 좋을 것 같았지만 중간부터 다 수정하기에는 시간적인 여유도 없어서 더 힘들 것이라 생각해서 다음부터는 그렇게 할 예정

- 하면서 많은 수정이 있었는데 중간오류들이 발생할 때가 정말 많았음
  - git branch를 잘 쓰고 싶었지만 잘 사용하지는 못했음
  - 이번에는 중요성을 깨달았고 다음부터는 잘 사용해보는 걸로
  - 중간마다 커밋을 하면서 잘 작동하는지 확인해보는 것도 중요한 듯
  - 만약에 고쳐서 안된다 싶으면 바로 이전 깃상태로 되돌리는 방식이 있었는데 git을 조금은 몰아서해서 잘 사용하지 못했던 것 같음


- 일주일 동안 프로젝트를 한번에 진행해보니 디테일을 잡는데는 상당히 오랜 시간이 걸리는 것을 느꼇고, 만들고 싶은 것을 만드려면 빨리 만들어지는 경우도 있지만 늘 새로 찾고 도전해보고 하는 것이 중요하다고 느꼈습니다. 앞으로도 늘 새로운 것을 찾기위해 머리를 잡아뜯겠지만 스트레스 받아도 그 스트레스를 해결으로 푸는 순간까지 열심히 해야겠다고 생각했습니다.
